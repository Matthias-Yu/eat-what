# 小家菜单

一个给家人使用的微信小程序：把日常点餐、购物车、订单记录和生活 TODO 放在同一个温暖的小空间里。

## 已实现

- 首页：动态问候、今日推荐、快捷入口、待办概览
- 点餐：分类筛选、关键词搜索、加购、数量调整、口味备注、下单反馈
- 待办：分类与状态筛选、新建、编辑、完成、长按删除、进度统计
- 我的：最近订单、完成统计、菜单管理、本地数据管理
- 持久化：购物车、订单、待办使用云端共享，本地存储作为离线缓存

## 家庭云空间

项目已连接云开发环境 `cloudbase-4gz52ssycf6b2383`，并部署 `familyApi` 云函数。家庭云空间分为「创建」与「加入」两步：创建者点击创建即可建立云空间并自动成为管理员，系统自动生成 6 位邀请码并显示出来；其他人凭邀请码加入即为普通成员，成员数量不限。管理员可在「成员管理」中查看全部成员、修改昵称或移除成员；成员也可修改自己的昵称。本地存储作为离线缓存，在线状态下每 12 秒自动拉取一次家庭数据。成员下单后会给管理员生成点餐通知；云函数已预留微信订阅消息发送通道，可通过 `ORDER_NOTICE_TEMPLATE_ID` 环境变量接入模板。

云端集合由函数首次运行时自动初始化：

- `family_users`：微信用户与家庭关系
- `family_households`：家庭成员和邀请码
- `family_data`：共享购物车、订单与待办

## 项目结构

```text
components/
  bottom-nav/       自定义底部导航
  navigation-bar/   小程序导航栏
data/
  menu.js           菜单与分类数据
pages/index/        主页面与交互
utils/
  date.js           日期与问候格式化
  storage.js        本地存储适配层
```

## 运行

用微信开发者工具导入当前目录即可运行。项目不依赖 npm 包，也不需要先启动后端。

## 后续接入后端

目前所有读写集中在 `utils/storage.js`。接入微信云开发时，可以保持页面数据结构不变，将该文件替换为云数据库仓储层，并把订单提交改为云函数调用。菜单数据也可以从 `data/menu.js` 平滑迁移到数据库集合。
