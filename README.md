# 小家菜单

一个给家人使用的微信小程序：把日常点餐、购物车、订单记录和生活 TODO 放在同一个温暖的小空间里。

## 已实现

- 首页：动态问候、今日推荐、快捷入口、待办概览
- 点餐：分类筛选、关键词搜索、加购、数量调整、口味备注、下单反馈
- 待办：分类与状态筛选、新建、编辑、完成、长按删除、进度统计
- 我的：最近订单、完成统计、菜单管理、本地数据管理
- 持久化：购物车、订单、待办使用云端共享，本地存储作为离线缓存

## 家庭云空间

项目已连接云开发环境 `cloudbase-4gz52ssycf6b2383`，并部署 `familyApi` 云函数。家庭云空间分为「创建」与「加入」两步：创建者点击创建即可建立云空间并自动成为管理员，系统自动生成 6 位邀请码并显示出来；其他人凭邀请码加入即为普通成员，成员数量不限。管理员可在「成员管理」中查看全部成员、修改昵称或移除成员；成员也可修改自己的昵称。本地存储作为离线缓存。在线状态下每 30 秒只检查轻量版本元数据，检测到变化后才拉取家庭数据。成员下单后会给管理员生成点餐通知；云函数已预留微信订阅消息发送通道，可通过 `ORDER_NOTICE_TEMPLATE_ID` 环境变量接入模板。

云端集合由函数首次运行时自动初始化：

- `family_users`：微信用户与家庭关系
- `family_households`：家庭成员和邀请码
- `family_data`：共享购物车、农场、花园、留言、信件及资源版本元数据
- `family_resources`：待办、订单、心愿、菜单、地点等高增长资源；旧家庭首次读取时自动迁移
- `family_visits`：成员进入记录，默认保留 90 天，界面仅展示脱敏用户标识
- `family_ai_usage`：AI 每日额度
- `family_rate_limits`：邀请码尝试限制和关键操作幂等锁

## 项目结构

```text
components/
  bottom-nav/       自定义底部导航
  navigation-bar/   小程序导航栏
data/
  menu.js           菜单与分类数据
pages/index/        主页面与交互
utils/
  date.js           日期与问候格式化
  storage.js        本地存储适配层
  home-model.js     首页、纪念日、留言和信件数据模型
  garden-model.js   农场与花园数据模型
  sync.js           云端版本比较
```

## 运行

用微信开发者工具导入当前目录即可运行。项目不依赖 npm 包，也不需要先启动后端。

## 校验

提交前运行 `npm run check`。测试覆盖存储键注册、数据模型清洗、并发合并、资源分片以及轻量同步版本判断。
